Como os maiores SaaS fazem upgrade/checkout — Pesquisa real

Patterns extraidos de Notion, Figma, Linear, Slack, Miro, Loom, Coda, Neon, HubSpot, Lovable e mais 20+ produtos

5 Patterns dominantes que os top SaaS usam

1. Modal de upgrade contextual (nao pagina separada)

Quem faz: Notion, Figma, Slack, Miro, HubSpot, Loom, Neon, Lovable
Como: Quando o usuario clica numa feature bloqueada ou atinge um limite, abre um modal sobre o app atual — nao uma pagina separada. O modal mostra o valor da feature + plano + CTA. O usuario nunca sai do contexto do trabalho.
Neon case study: Redesenharam de multi-step page pra um unico modal clean. Resultado: +7% conversao, -47% abandono.

2. Action-based triggers (nao calendar-based)

Quem faz: Miro, Grammarly, Spotify, Dropbox, Slack, Coda
Como: O upgrade nao aparece porque "faltam 3 dias". Aparece quando o usuario tenta usar algo bloqueado ou atinge um limite. Miro mostra uma animacao do que a feature faz e entao oferece upgrade. Grammarly mostra issues detectadas que so o premium resolve. Spotify bloqueia skip e mostra o valor do premium.
Dado: Feature-gated triggers convertem mais que calendar-based (GrowthLayer, 2026).

3. Reverse trial (dar premium, depois tirar)

Quem faz: Coda, Duolingo, Canva, varias startups AI
Como: O usuario recebe acesso total ao plano premium durante X dias. Quando acaba, volta pro free — mas ja "sentiu" o que perde. Coda: apos upgrade trial, destaca visualmente na UI tudo que e "feature do trial" com badges. Quando o trial acaba, o usuario sabe exatamente o que perde.
Dado: Endowment effect — usuarios valorizam 2-3x mais algo que ja "possuem" (Kahneman).

4. Checkout via Stripe hosted ou 1-click

Quem faz: Linear, Vercel, v0, Better Stack, Figma, Notion
Como: Nao constroem form de pagamento custom. Usam Stripe Checkout hosted — redirect pra pagina Stripe, preenche cartao, volta pro app. Ou usam Stripe Elements embeddados num modal minimo. Zero fricao, zero custom code desnecessario.
Figma: Um botao "Complete purchase" → Stripe processa → volta pro app. Sem 4 steps. Linear: Pricing page → "Get started" → Stripe checkout → done.

5. Profile/Settings como hub de upgrade

Quem faz: Lovable, Linear, Figma, Slack
Como: O upgrade nao e um fluxo separado com paginas proprias. Vive dentro de Settings → Billing. Lovable transforma o perfil do usuario no ponto principal de nudges e upsells. Linear tem tudo em Settings → Workspace → Plans. O usuario sente que esta gerenciando sua conta, nao "sendo vendido".
Insight: Nenhum dos top SaaS tem um "checkout flow" de 4 paginas dedicadas. O upgrade e integrado ao produto.

Conclusao critica pra voce

O que o ChatFunnel faz hoje (4 paginas dedicadas de checkout) nao e o que nenhum top SaaS faz. O pattern dominante e: modal contextual + Stripe checkout/hosted + settings page. Ninguem cria um "wizard de 4 steps" separado do app.

A pergunta real nao e "como redesenhar os 4 steps" — e "deveriamos ter steps?"

3 direcoes possiveis baseadas na pesquisa real

A

Modal contextual + Stripe Checkout hosted

Quando o trial acaba (ou o usuario tenta feature premium): abre modal no app mostrando metricas do trial + planos. Ao clicar "Selecionar plano" → redirect pra Stripe Checkout hosted. Stripe cuida de tudo (form, seguranca, 3DS). Volta pro app apos pagamento. Sem nenhuma pagina custom de checkout.

Pattern de: Linear, Vercel, Better Stack, v0

B

Upgrade integrado em Settings + banner no app

Banner persistente no topo do app ("Trial expira em X dias"). Clicando, vai pra Settings → Planos onde ve metricas, planos e paga via Stripe Elements embeddado. O upgrade e parte do app, nao uma experiencia separada. Sem paginas dedicadas.

Pattern de: Slack, Figma, Lovable, Notion

C

Hibrido: Banner + Modal rico + Stripe hosted

Banner no app → clica → abre modal full-screen (overlay) com: metricas do trial hero, cards de plano, social proof, tudo numa unica tela overlay. Ao selecionar plano → redirect Stripe Checkout. Junta o melhor de A (simplicidade) com contexto visual (metricas, social proof). Nao e uma pagina separada, e um overlay sobre o app.

Pattern de: Neon (modal redesign), HubSpot, Coda
