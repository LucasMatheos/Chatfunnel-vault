## Decisoes

- Aprovado design B (Header com Icone) para IntegrationsCard
- Aprovado layout C (Split Panel) para modal ConfigureMcp
- DialogControl nao serve para layouts split edge-to-edge — usar Dialog primitivos
- Skeleton loading adotado como estrategia padrao de carregamento (regra adicionada ao CLAUDE.md do front)
- Preferencia Vee para validacao, InputControl quando nao precisa de validacao (regra adicionada ao CLAUDE.md do front)
- ConfigureMcp refatorado em pasta com subcomponentes (padrao do projeto para componentes complexos)
- Import com path explicito /index.vue (padrao do projeto, Vite nao resolve pasta automaticamente)
- Tabs underline extraidas em componente reutilizavel TabUnderline (shadcn-custom)
- Card do DS nao cabe para itens pequenos de lista (over-engineering) — manter divs com classes inline
- Badge ganhou variantes square e non-uppercase para cobrir mais casos
- OrganizationForm: StepperControl com 2 steps (dados + plano) em dialog XL
- Apresentacao de planos: estilo Notion com highlight "Mais escolhidos" e link para modal de comparacao

## Bloqueios

- Plugin codex nao carregou apos instalacao — precisou recarregar sessao (Unknown skill: codex:setup)
- CORS bloqueava requests do frontend (:5173) para o MCP (:8000) — resolvido com app.enableCors()
- Reka UI nao exporta AlertDialogRootProps/AlertDialogRootEmits — precisou inlinar tipos
- Hooks quebravam em sub-repos por usar caminhos relativos — corrigido para absolutos
- Erro "Selected account is not allowed by token" na autenticacao MCP — investigado mas nao resolvido explicitamente no raw
